This directory holds annotated captures of the on-device flows
described in [PROPOSAL.md](../PROPOSAL.md) §5.

`LlamaCppAdapter` running `Bonsai-1.7B` (PrismML llama.cpp
fork). Every visible affordance — bilingual translation, smart
reply, summary, insights, task extraction, metrics — is real
on-device LLM output, with the privacy strip showing
`compute: on-device`, `model: bonsai-1.7b`, `egress: 0 B`.

## How to reproduce

1. **Build llama-server** (PrismML fork):

   ```bash
   git clone -b prism https://github.com/kennguy3n/llama.cpp ~/llama-cpp-prism
   cd ~/llama-cpp-prism
   cmake -B build -DGGML_NATIVE=ON -DLLAMA_CURL=OFF -DCMAKE_BUILD_TYPE=Release
   cmake --build build -j$(nproc) --target llama-server
   ```

2. **Download the model** (~237 MB):

   ```bash
   curl -L -o models/Bonsai-1.7B.gguf \
     https://huggingface.co/prism-ml/Bonsai-1.7B-gguf/resolve/main/Bonsai-1.7B.gguf
   ```

3. **Start `llama-server`** with single-slot interactive settings:

   ```bash
   ~/llama-cpp-prism/build/bin/llama-server \
     -m models/Bonsai-1.7B.gguf \
     -c 4096 --parallel 1 -t 4 --host 127.0.0.1 --port 11400
   ```

4. **Start the Ollama-API shim** (translates `/api/generate` →
   `/completion` and strips `<think>` tags):

   ```bash
   python3 scripts/ollama-shim.py --port 11434 --upstream http://127.0.0.1:11400
   ```

5. **Start the Go backend and Electron app**:

   ```bash
   cd backend && go run ./cmd/server
   cd frontend && OLLAMA_BASE_URL=http://127.0.0.1:11434 \
     MODEL_NAME=bonsai-1.7b npm run electron:dev
   ```

6. Open the B2C surface (`ch_dm_alice_minh` is auto-selected) and
   wait for every translation card to finish rendering before
   capturing each screenshot.

For each capture, verify three things before clicking shutter:

- The privacy strip shows `compute: on-device` and `egress: 0 B`.
- The model name is `bonsai-1.7b` (or the alias bound to your local
  GGUF), **not** `[MOCK]`.
- There are no streaming indicators (`Translating on-device…`,
  `Generating…`) anywhere in the frame.
