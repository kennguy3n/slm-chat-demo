## Phase 6 — Confidential server mode and enterprise hardening

- [x] Confidential server compute mode — new `ConfidentialServerAdapter` (`frontend/electron/inference/confidential-server.ts`) implementing the `Adapter` contract with NDJSON streaming against a configurable `CONFIDENTIAL_SERVER_URL` (default `http://localhost:8090`); refuses with a clear "unreachable" error when the server is down (no silent fallback). Router extended to a third `'server'` tier with `attachServer()` / `hasServer()` and a policy-gated `decide()` that requires both adapter availability AND `CONFIDENTIAL_SERVER_POLICY=allow`. Bootstrap (`bootstrap.ts`) probes `CONFIDENTIAL_SERVER_URL/v1/health` on startup and only wires the server tier when both the ping resolves and the policy permits it. `model:status` IPC reports `serverModel`, `serverAvailable`, `serverUrl`; the `DeviceCapabilityPanel` renders a "Confidential server" sub-section behind that flag. Tests: `confidential-server.test.ts` (7 cases covering happy-path, unreachable, non-2xx, NDJSON streaming, ping), `router.test.ts` confidential-server-tier suite (refusal modes, policy gating, run/stream wiring), `bootstrap.test.ts` confidential-server-probe suite, and Vitest panel-rendering coverage in `DeviceCapabilityPanel.test.tsx` and `PrivacyStrip.test.tsx`.
- [x] Redaction/tokenization before data egress — new `RedactionEngine` (`frontend/electron/inference/redaction.ts`) with `tokenize` (reversible — replaces emails / phones / SSNs / two-word names with `[EMAIL_n]` / `[PHONE_n]` / `[SSN_n]` / `[NAME_n]` placeholders and stores the mapping), `redact` (non-reversible `[REDACTED]` substitution), and `detokenize` (longest-first replacement so `[EMAIL_10]` doesn't get clobbered by `[EMAIL_1]`). The `InferenceRouter`'s `run` and `stream` paths tokenize the prompt before dispatching to the server adapter and detokenize the response (or each stream delta) before returning to the renderer, so the server only ever sees placeholder tokens. The `RedactionPolicy` carries fine-grained boolean flags plus a `customPatterns` escape hatch. UTF-8 byte length (`utf8ByteLength`) drives the egress-bytes counter so reported bytes match wire bytes. `PrivacyStrip` renders a new "Redaction" row ("3 items redacted (2 names, 1 email)") only for confidential-server outputs. Tests: `redaction.test.ts` (15 cases covering detection, tokenize/detokenize round-trip, selective policies, custom patterns, UTF-8 byte counting) and an integration assertion in `router.test.ts` that the server stub never sees the original PII.
- [x] Data egress summary display — new `EgressTracker` singleton (`frontend/electron/inference/egress-tracker.ts`) with `record({ timestamp, taskType, egressBytes, redactionCount, model, channelId })`, `summary()` (totals + `byChannel` + `byModel` + newest-first `recent`), `reset()`. The router records into the global tracker on every server-routed inference. New IPC channels `egress:summary` / `egress:reset` are exposed on the preload bridge as `window.electronAI.egressSummary()` / `egressReset()`. Renderer side: new `EgressSummaryPanel` component renders total bytes (with a prominent "0 B" zero-state), per-channel and per-model breakdowns, a recent-activity timeline, and a Reset button; the existing TopBar "Egress" badge now reads from the live tracker via the new `useEgressSummary` hook. Tests: `egress-tracker.test.ts` (zero state, aggregation by channel/model, ordering, reset), `EgressSummaryPanel.test.tsx` (zero state, populated rendering, reset interaction, channelless grouping).
- [x] EgressSummaryPanel + useEgressSummary hook + formatEgressBytes utility — three new renderer-side files (`frontend/src/features/ai/EgressSummaryPanel.tsx`, `useEgressSummary.ts`, `formatEgressBytes.ts`) forming the egress UI: prominent "0 B" zero-state, per-channel and per-model breakdowns, recent-activity timeline, Reset button, and a human-readable byte formatter.
